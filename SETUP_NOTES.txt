Environment setup (no administrator rights)
------------------------------------------

1. Install Python 3.10+ for your user only (does not require admin):
   - Download the Windows installer from https://www.python.org/downloads/
   - Run it and choose "Install for current user" / uncheck "Install for all users"
   - Enable "Add python.exe to PATH" if offered

2. Open a new terminal and verify:
   py -3.12 --version
   (or python --version pointing to 3.10+)

3. From the smoke-plume repo folder:
   py -3.12 -m venv .venv
   .\.venv\Scripts\Activate.ps1
   python -m pip install --upgrade pip
   pip install -r requirements.txt

If the silent installer was run without admin, Python may be under:
%LOCALAPPDATA%\Programs\Python\Python312\python.exe

Large rasters (*.tif, *.nc) and pdf-pages/ are gitignored; keep data local.

GDAL / OSGeo (optional; keep separate from .venv)
-----------------------------------------------
- Do **not** `pip install gdal` (or OSGeo Python bindings) into this project’s `.venv`.
  Use the standalone GDAL tools from **OSGeo4W** or **QGIS** (or conda env), or use
  `scripts/tempo_l2_to_4326.py` with the venv (rasterio/scipy only; no extra pip).
- To use OSGeo’s `gdalwarp`/`gdalinfo`, open **“OSGeo4W Shell”** (or add
  `C:\OSGeo4W\bin` to PATH for that session) so `gdalwarp.exe` resolves. Subdataset
names for TEMPO `.nc` files depend on the GDAL driver; `gdalinfo` on the file
lists them. Swath → regular 4326 often uses `gdalwarp -geoloc` once lat/lon
arrays are exposed the way your GDAL build expects.

Run analysis (repo root; use `.venv\Scripts\python.exe` if venv not activated)
----------------------------------------------------------------------------
  python scripts/tempo_l2_to_4326.py --nc data/palisades/tempo/<granule>.nc
  python scripts/palisades_pipeline.py --write-maps
  python scripts/palisades_sanity_check.py
Outputs go under `results/palisades/` (gitignored).
